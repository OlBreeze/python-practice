# Домашнє завдання: Django Ninja API Projects

## 1. API для керування завданнями (Task Manager)

Розробити API для керування завданнями, де користувачі можуть створювати, оновлювати, видаляти та переглядати завдання.

### Основні функції:

- CRUD операції (створення, читання, оновлення, видалення) для завдань.
- Фільтрування завдань за статусом (наприклад, "виконано", "не виконано").
- Сортування завдань за датою створення або терміном виконання.
- Облік авторизації користувача.

### Візуалізація:

- Використання Swagger UI для відображення документації API.
- Приклад даних можна відобразити через Postman для тестування API.

---

## 2. API для онлайн-магазину (E-commerce)

Розробити API для управління продуктами та замовленнями в інтернет-магазині.

### Основні функції:

- CRUD операції для товарів.
- Операції для додавання/видалення товарів у кошик.
- Оформлення замовлень із вибором товарів із кошика.
- Управління статусом замовлень (наприклад, у процесі, відправлений, доставлений).

### Візуалізація:

- Документація API через Swagger UI
- Використання Postman для надсилання запитів та тестування функціоналу API.

---

## 3. API для керування колекцією фільмів (Movie Collection)

Створити API для керування колекцією фільмів, включаючи операції з фільмами, їх жанрами, рейтингами та відгуками.

### Основні функції:

- CRUD операції для фільмів
- Додавання жанрів та відгуків.
- Можливість фільтрації фільмів за жанрами, рейтингом та датою випуску.
- Можливість пошуку фільмів за частковим збігом назви.

### Візуалізація:

- Swagger UI для зручного відображення всієї документації.
- Тестування API за допомогою Postman.

---

## 4. API для блогу (Blog Platform)

Розробити REST API для керування постами в блозі.

### Основні функції:

- CRUD операції для постів.
- Можливість додавання коментарів.
- Управління тегами для кожного посту.
- Аутентифікація користувачів для написання та редагування постів.

### Візуалізація:

- Документація API через Swagger UI
- Приклад тестування за допомогою Postman, включаючи запити для аутентифікації.

---

## 5. API для моніторингу стану серверів (Server Monitoring)

Створити API для моніторингу стану серверів та їх метрик.

### Основні функції:

- CRUD операції для додавання та видалення серверів.
- Відстеження стану серверів (увімкнений/вимкнений) та метрик (наприклад, навантаження, пам'ять, процесор).
- Надсилання повідомлень при досягненні критичних значень метрик.

### Візуалізація:

- Візуалізація метриків за допомогою Swagger UI.
- Надсилання запитів через Postman для моніторингу серверів.

---

## 6. API для керування книгами (Book Library)

Розробити API для керування бібліотекою книг.

### Основні функції:

- CRUD операції для додавання/видалення книг.
- Пошук книг за назвою, автором або жанром.
- Підтримка користувачів, які можуть брати книги на час (оренда).

### Візуалізація:

- Swagger UI для тестування та відображення API.
- Приклад роботи з API через Postman.

---

## 7. API для управління студентами та курсами (Student Course Management)

Створити API для управління студентами, курсами та результатами іспитів.

### Основні функції:

- CRUD операції для студентів та курсів.
- Реєстрація студентів на курси.
- Оцінка студентів з урахуванням їх результатів.
- Підрахунок середньої оцінки за кожним курсом.

### Візуалізація:

- Приклад роботи з API через Postman.
- Документація та тестування API за допомогою Swagger UI.

---

## P.S.

Для кожного з проектів передбачити аутентифікацію користувачів, тобто усі CRUD-операції можуть здійснювати тільки зареєстровані користувачі (використовуйте декоратори).
